**The dashboard layout**

1. Selectors (top, compact vertically).
2. Plots (middle, compact vertically).
3. Details (bottom).

First, you need to take all the `.task_id` values found for all models and merge them into a single set, then convert to a list. We will refer to this list as `IDs`. The required selectors:

- Task: to select one of the IDs. Should be printed in form f'#{i} ({id})', where `id` is the task id, and `i` is the position in the IDs. Thus, `i` is an incremental counter from 0 to `len(IDs)`.
- Model: select one of the models we have, such as "qwen_7b".
- Plot metric: select one of the metric names listed in `GenerationTaskMetrics` (only those which are float-valued and not list[float]) for plots.
- Analysis: a combination of a title+model_name found in .analysis in at least one of the GenerationTaskAnalysis objects.

**The dashboard callbacks**

The layout should have only a single callback that processes the described selectors and fill "plots" and "details" areas.

**1. How to fill "plots".** Here we aim to visualize the selected metric for all the data we have. A plot is a thin and wide image like plt.matshow or analog. The horizontal (long) axis is a task index (`i` in the task selector), and the vertical (typically short) axis is a model name. So, each cell is associated with a single `GenerationTaskAnalysis` object, or empty (if such task id is not given for the given model). The value in the cell is the "Plot metric" (from the selector) for that obect, or white (ef empty).

**2. How to fill "details".** If the given model does NOT have the prediction for a given task, return. Otherwise, display the information from the corresponding `GenerationTaskAnalysis` object:

- Task id, answerability, multi_turn, question_type in the header of the "details" block.
- Dialog and reference in the left side.
- Documents, prediction, analysis prompt and answer in the right side (if the given title+model_name exists in the current GenerationTaskAnalysis object.
- Metrics in the footer.

**Several important notes** about the dashboard:

- Define styles in separate variables to make the layout code shorter.
- Use python3.12 typing (i.e. `list` instead of `List`).
- The layout need to be compact if possible.
- When visualizing a .dialog, highlight "User:" and "Assistant:" substrings in bold.
